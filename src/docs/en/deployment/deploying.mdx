# Deploying

docs-template is a standard Next.js application and can be deployed to any platform that supports Next.js.

---

## Pre-build Scripts

Two scripts must be considered before deploying to production.

### Search index

The full-text search index is generated automatically by `bun build`, so no extra step is required for most deployments.

### Git metadata

To display accurate last-modified dates and author information on each page, run the Git metadata script **before** the build:

```bash copy="true"
bun run generate-git-metadata && bun run build
```

<Callout type="info">
Lint-staged integration

`generate-git-metadata` also runs automatically via lint-staged on every commit, keeping metadata up to date as you work.
</Callout>

---

## Deployment Platforms

### Vercel

Import the repository on [vercel.com](https://vercel.com) and deploy with zero configuration. Vercel auto-detects Next.js and sets the correct build command.

To include Git metadata, set the build command in the Vercel dashboard:

```bash copy="true"
bun run generate-git-metadata && bun run build
```

### Netlify

Use the [Next.js on Netlify](https://docs.netlify.com/frameworks/next-js/overview/) build plugin. Set the build command to:

```bash copy="true"
bun run generate-git-metadata && bun run build
```

### Self-hosted

Run the production build on any Node.js 20+ server:

```bash copy="true"
bun run generate-git-metadata && bun run build
bun start
```

The server starts on port 4000 by default.

<Callout type="tip">
Environment variables

If your deployment platform clones a shallow Git history, the `generate-git-metadata` script may not have access to full commit history. Use `git fetch --unshallow` in your CI pipeline to ensure accurate metadata.
</Callout>
