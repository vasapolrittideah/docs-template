# Installation

This guide walks you through cloning the repository, installing dependencies, and running the development server locally.

---

## Prerequisites

Before you begin, make sure you have the following installed:

- **Node.js 20+** — [Download](https://nodejs.org/)
- **Bun** (recommended) — [Download](https://bun.sh/)

<Callout type="info">
Why Bun?

The project uses Bun as its package manager and script runner. While npm or pnpm may work, all scripts and lockfiles are optimized for Bun.
</Callout>

---

## Setup

### 1. Clone the repository

```bash copy="true"
git clone https://github.com/your-username/docs-template.git
cd docs-template
```

Alternatively, click **"Use this template"** on the repository page on GitHub to create a new repository from this template.

<Callout type="warning">
Internal use only

This repository is maintained exclusively for TCC Technology internal projects. Do not use it for public or third-party projects.
</Callout>

### 2. Install dependencies

```bash copy="true"
bun install
```

### 3. Configure environment variables

Copy the example environment file and fill in the required values:

```bash copy="true"
cp .env.example .env.local
```

See the [Authentication](./authentication) page for a full description of each environment variable.

### 4. Start the development server

```bash copy="true"
bun dev
```

Open [http://localhost:4000](http://localhost:4000) in your browser. The `bun dev` command automatically generates the full-text search index before starting the server.

---

## Available Scripts

| Script | Description |
| ------ | ----------- |
| `bun dev` | Generate search index, then start the development server on port 4000 |
| `bun build` | Generate search index, then build for production |
| `bun start` | Start the production server on port 4000 |
| `bun lint` | Run ESLint across the project |
| `bun run generate-search-index` | Build the full-text search index from MDX content |
| `bun run generate-git-metadata` | Pre-generate Git last-modified metadata for all doc pages |

---

## Pre-build Scripts

Two scripts prepare static assets that the site depends on. Both are called automatically by `bun dev` and `bun build`, but you can also run them manually.

### Search index

Scans all MDX files and writes JSON search indexes to `public/`:

```bash copy="true"
bun run generate-search-index
```

### Git metadata

Reads `git log` to collect last-modified dates and authors for every doc page. The output is written to `public/git-metadata.json`:

```bash copy="true"
bun run generate-git-metadata
```

<Callout type="tip">
Automatic on commit

`generate-git-metadata` is wired to **lint-staged** and runs automatically on every `git commit`, so the metadata file stays current without any manual steps.
</Callout>
