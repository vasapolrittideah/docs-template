# Adding Pages

Documentation pages in docs-template are MDX files stored under `src/docs/[locale]/[group]/[slug].mdx`. Adding a new page takes two steps: creating the MDX file and registering it in the navigation.

---

## File Location

All MDX files follow this path convention:

```text
src/docs/<locale>/<group>/<slug>.mdx
```

- `<locale>` — `en` or `th` (or any locale you have configured).
- `<group>` — A folder that maps to a navigation group in the sidebar.
- `<slug>` — The page identifier used in the URL.

A page at `src/docs/en/guides/configuration.mdx` is served at `/en/docs/guides/configuration`.

---

## Creating a Page

### 1. Create the MDX file

Create a file at `src/docs/[locale]/[group]/[slug].mdx`. Every page must start with a metadata export:

```mdx:src/docs/en/guides/configuration.mdx copy="true"
export const metadata = {
  title: 'Configuration | My Docs',
  description: 'How to configure the project.',
}

# Configuration

Your content goes here.
```

<Callout type="info">
Create for all locales

Always create an MDX file for **each locale** you support (e.g., both `en` and `th`) to avoid 404 errors when users switch languages.
</Callout>

### 2. Register in navigation

Open `src/docs/[locale]/navigation.json` for each locale and add a `{ "slug", "title" }` entry to the relevant group's `pages` array:

```json:src/docs/en/navigation.json copy="true"
[
  {
    "slug": "guides",
    "title": "Guides",
    "pages": [
      { "slug": "configuration", "title": "Configuration" }
    ]
  }
]
```

The page is now accessible in the sidebar.

---

## Adding a New Group

If no existing group fits your page, add a new group object to the top-level array in both locale navigation files:

```json copy="true"
[
  {
    "slug": "getting-started",
    "title": "Getting Started",
    "pages": [
      { "slug": "introduction", "title": "Introduction" }
    ]
  },
  {
    "slug": "guides",
    "title": "Guides",
    "pages": [
      { "slug": "configuration", "title": "Configuration" }
    ]
  }
]
```

<Callout type="warning">
Avoid duplicate slugs

Page slugs must be unique within a group. Two pages with the same slug in the same group will cause routing conflicts.
</Callout>
