# MDX Components

docs-template maps standard Markdown elements to styled components automatically. A small set of additional components ‚Äî like `Callout` ‚Äî are available via import for richer content.

## Auto-mapped Elements

The following elements require no import and are styled automatically:

| Element | MDX syntax |

| Headings | `#`, `##`, `###`, ‚Ä¶ |
| Paragraphs | Plain text |
| Lists | `- item` / `1. item` |
| Tables | `\| col \| col \|` |
| Inline code | `` `code` `` |
| Code blocks | ` ``` ` fenced blocks |
| Links | `[text](url)` |
| Dividers | `---` |

## Callout

Use `Callout` to highlight important information. Import it at the top of your MDX file:

```mdx copy="true"
import Callout from '@/components/mdx/callout';
```

### Usage

```mdx copy="true"
<Callout type="info">
Callout Title

Body text providing additional context.
</Callout>
```

The first child is rendered as the **title**; subsequent children are rendered as **body content**.

### Types

| Type | When to use |
|---|---|
| `info` | Setup notes, general context |
| `tip` | Best practices, productivity hints |
| `warning` | Potential issues, non-destructive cautions |
| `danger` | Breaking changes, destructive or irreversible actions |

<Callout type="info">
Info callout

Use this for setup notes or general context that readers should be aware of.
</Callout>

<Callout type="tip">
Tip callout

Use this for best practices and recommendations.
</Callout>

<Callout type="warning">
Warning callout

Use this to flag potential issues or configuration pitfalls.
</Callout>

<Callout type="danger">
Danger callout

Use this for breaking changes or irreversible actions.
</Callout>

### Collapsible callouts

Pass `isCollapsed` to render the callout collapsed by default:

```mdx copy="true"
<Callout type="warning" isCollapsed>
Advanced Details

This content is hidden until the user expands it.
</Callout>
```

## Code Blocks

Fenced code blocks support syntax highlighting via Shiki. Add props directly on the opening fence line.

### Copy button

Add `copy="true"` to display a copy-to-clipboard button:

````mdx copy="true"
```bash copy="true"
bun install
```
````

### File path label

Append `:path/to/file` to the language identifier to show a file path header above the block:

````mdx copy="true"
```ts:src/lib/example.ts copy="true"
const greeting = 'Hello, world!';
```
````

<Callout type="info">
Language and filepath

The text after `:` is rendered as a label only ‚Äî it does not affect syntax highlighting. Use the correct language identifier before `:` for proper highlighting.
</Callout>
