# การติดตั้ง

คู่มือนี้จะแนะนำขั้นตอนการ Clone Repository, ติดตั้ง Dependencies และรัน Development Server บนเครื่องของคุณ

---

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มต้น ตรวจสอบให้แน่ใจว่าคุณได้ติดตั้งสิ่งต่อไปนี้แล้ว:

- **Node.js 20+** — [ดาวน์โหลด](https://nodejs.org/)
- **Bun** (แนะนำ) — [ดาวน์โหลด](https://bun.sh/)

<Callout type="info">
ทำไมต้อง Bun?

โปรเจกต์นี้ใช้ Bun เป็น Package Manager และ Script Runner แม้ว่า npm หรือ pnpm อาจใช้งานได้ แต่ Scripts และ Lockfiles ทั้งหมดได้รับการปรับแต่งสำหรับ Bun
</Callout>

---

## การตั้งค่า

### 1. Clone Repository

```bash copy="true"
git clone https://github.com/your-username/docs-template.git
cd docs-template
```

หรือคลิก **"Use this template"** บนหน้า Repository ใน GitHub เพื่อสร้าง Repository ใหม่จากเทมเพลตนี้

<Callout type="warning">
สำหรับใช้งานภายในเท่านั้น

Repository นี้ดูแลรักษาเฉพาะสำหรับโปรเจกต์ภายในของ TCC Technology ห้ามใช้สำหรับโปรเจกต์สาธารณะหรือบุคคลภายนอก
</Callout>

### 2. ติดตั้ง Dependencies

```bash copy="true"
bun install
```

### 3. ตั้งค่า Environment Variables

คัดลอกไฟล์ Environment ตัวอย่างและกรอกค่าที่จำเป็น:

```bash copy="true"
cp .env.example .env.local
```

ดูหน้า [Authentication](./authentication) สำหรับคำอธิบายเต็มของ Environment Variable แต่ละตัว

### 4. เริ่มต้น Development Server

```bash copy="true"
bun dev
```

เปิด [http://localhost:4000](http://localhost:4000) ในเบราว์เซอร์ คำสั่ง `bun dev` จะสร้าง Search Index แบบ Full-text โดยอัตโนมัติก่อนเริ่ม Server

---

## Scripts ที่ใช้ได้

| Script | คำอธิบาย |
| ------ | --------- |
| `bun dev` | สร้าง Search Index แล้วเริ่ม Development Server บน Port 4000 |
| `bun build` | สร้าง Search Index แล้ว Build สำหรับ Production |
| `bun start` | เริ่ม Production Server บน Port 4000 |
| `bun lint` | รัน ESLint ทั่วทั้งโปรเจกต์ |
| `bun run generate-search-index` | สร้าง Search Index แบบ Full-text จากเนื้อหา MDX |
| `bun run generate-git-metadata` | สร้าง Git Last-modified Metadata ล่วงหน้าสำหรับทุกหน้าเอกสาร |

---

## Scripts ก่อน Build

Scripts สองตัวเตรียม Static Assets ที่เว็บไซต์ต้องการ ทั้งสองจะถูกเรียกโดยอัตโนมัติด้วย `bun dev` และ `bun build` แต่คุณสามารถรันด้วยตนเองได้เช่นกัน

### Search Index

สแกนไฟล์ MDX ทั้งหมดและเขียน JSON Search Indexes ไปยัง `public/`:

```bash copy="true"
bun run generate-search-index
```

### Git Metadata

อ่าน `git log` เพื่อรวบรวมวันที่แก้ไขล่าสุดและผู้เขียนสำหรับทุกหน้าเอกสาร ผลลัพธ์จะถูกเขียนไปยัง `public/git-metadata.json`:

```bash copy="true"
bun run generate-git-metadata
```

<Callout type="tip">
อัตโนมัติเมื่อ Commit

`generate-git-metadata` เชื่อมต่อกับ **lint-staged** และรันโดยอัตโนมัติทุกครั้งที่ `git commit` ดังนั้นไฟล์ Metadata จะทันสมัยอยู่เสมอโดยไม่ต้องดำเนินการด้วยตนเอง
</Callout>
