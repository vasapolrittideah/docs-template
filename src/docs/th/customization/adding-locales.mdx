# การเพิ่มภาษา

docs-template มาพร้อมกับภาษาอังกฤษ (`en`) และภาษาไทย (`th`) หน้านี้อธิบายวิธีเพิ่ม locale ใหม่ให้กับเว็บไซต์

---

## ขั้นตอน

### 1. ลงทะเบียน locale ใน routing config

เพิ่ม locale identifier ใหม่ในอาร์เรย์ `locales` ใน `src/i18n/routing.ts` และอาจอัปเดต `defaultLocale` ด้วย:

```ts:src/i18n/routing.ts copy="true"
import { defineRouting } from 'next-intl/routing';

export const routing = defineRouting({
  locales: ['en', 'th', 'ja'],
  defaultLocale: 'en',
});
```

### 2. สร้างไฟล์ messages

สร้าง `messages/[locale].json` และเพิ่มค่าที่แปลแล้วสำหรับทุก key ที่มีอยู่ใน `messages/en.json`:

```bash copy="true"
cp messages/en.json messages/ja.json
```

จากนั้นแทนที่ค่าใน `messages/ja.json` ด้วยข้อความที่แปลแล้ว

<Callout type="warning">
ต้องมีครบทุก key

ทุก key ที่มีอยู่ในไฟล์ messages ภาษาอังกฤษต้องมีอยู่ในไฟล์ locale ใหม่ด้วย key ที่ขาดหายไปจะทำให้เกิด runtime error เมื่อผู้ใช้สลับไปยัง locale นั้น
</Callout>

### 3. สร้างโฟลเดอร์ docs และ navigation

สร้างไฟล์ navigation และเนื้อหา MDX สำหรับ locale ใหม่:

```bash copy="true"
mkdir -p src/docs/ja
cp src/docs/en/navigation.json src/docs/ja/navigation.json
```

แปลค่า `title` ใน `src/docs/ja/navigation.json` แล้วสร้างไฟล์ MDX ที่ตรงกันใน `src/docs/ja/[group]/[slug].mdx`

<Callout type="tip">
Slug ยังคงเหมือนเดิม

รักษาค่า `slug` ให้เหมือนกันในไฟล์ navigation ทุก locale เปลี่ยนเฉพาะค่า `title` และเนื้อหา MDX เท่านั้น
</Callout>

---

## ผลลัพธ์

หลังจากทำขั้นตอนเหล่านี้เสร็จ locale ใหม่จะพร้อมใช้งานที่ `/ja/docs/...` และจะปรากฏใน locale switcher โดยอัตโนมัติ
