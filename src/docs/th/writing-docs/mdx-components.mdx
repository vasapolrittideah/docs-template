# คอมโพเนนต์ MDX

docs-template แมป element Markdown มาตรฐานกับคอมโพเนนต์ที่มี style โดยอัตโนมัติ นอกจากนี้ยังมีคอมโพเนนต์เพิ่มเติม เช่น `Callout` สำหรับเนื้อหาที่หลากหลายขึ้น

---

## Element ที่แมปอัตโนมัติ

Element เหล่านี้ไม่ต้อง import และมี style โดยอัตโนมัติ:

| Element | MDX syntax |
|---|---|
| Heading | `#`, `##`, `###`, … |
| ย่อหน้า | ข้อความทั่วไป |
| รายการ | `- รายการ` / `1. รายการ` |
| ตาราง | `\| คอลัมน์ \| คอลัมน์ \|` |
| Inline code | `` `code` `` |
| Code block | ` ``` ` fenced blocks |
| ลิงก์ | `[ข้อความ](url)` |
| เส้นคั่น | `---` |

---

## Callout

ใช้ `Callout` เพื่อเน้นข้อมูลสำคัญ Import ที่ด้านบนของไฟล์ MDX:

```mdx copy="true"
import Callout from '@/components/mdx/callout';
```

### การใช้งาน

```mdx copy="true"
<Callout type="info">
ชื่อ Callout

ข้อความเพิ่มเติมที่ให้บริบท
</Callout>
```

child ตัวแรกจะแสดงเป็น**ชื่อ** ส่วน child ถัดไปจะแสดงเป็น**เนื้อหา**

### ประเภท

| ประเภท | เมื่อใช้ |
|---|---|
| `info` | หมายเหตุการตั้งค่า บริบททั่วไป |
| `tip` | แนวทางปฏิบัติที่ดี คำแนะนำ |
| `warning` | ปัญหาที่อาจเกิดขึ้น ข้อควรระวัง |
| `danger` | การเปลี่ยนแปลงที่ทำให้เสียหาย หรือการกระทำที่ย้อนกลับไม่ได้ |

<Callout type="info">
Info callout

ใช้สำหรับหมายเหตุการตั้งค่าหรือบริบทที่ผู้อ่านควรทราบ
</Callout>

<Callout type="tip">
Tip callout

ใช้สำหรับแนวทางปฏิบัติที่ดีและคำแนะนำ
</Callout>

<Callout type="warning">
Warning callout

ใช้เพื่อแจ้งปัญหาที่อาจเกิดขึ้นหรือข้อควรระวังในการตั้งค่า
</Callout>

<Callout type="danger">
Danger callout

ใช้สำหรับการเปลี่ยนแปลงที่ทำให้เสียหายหรือการกระทำที่ย้อนกลับไม่ได้
</Callout>

### Callout แบบยุบได้

ส่ง `isCollapsed` เพื่อให้ callout ยุบอยู่โดยค่าเริ่มต้น:

```mdx copy="true"
<Callout type="warning" isCollapsed>
รายละเอียดขั้นสูง

เนื้อหานี้จะซ่อนอยู่จนกว่าผู้ใช้จะขยาย
</Callout>
```

## Code Block

Fenced code block รองรับ syntax highlighting ผ่าน Shiki เพิ่ม prop ตรงในบรรทัดเปิด fence ได้เลย

### ปุ่มคัดลอก

เพิ่ม `copy="true"` เพื่อแสดงปุ่มคัดลอกไปยัง clipboard:

````mdx copy="true"
```bash copy="true"
bun install
```
````

### ป้ายชื่อไฟล์

เพิ่ม `:path/to/file` ต่อท้าย language identifier เพื่อแสดง header พร้อม path ไฟล์เหนือ block:

````mdx copy="true"
```ts:src/lib/example.ts copy="true"
const greeting = 'Hello, world!';
```
````

<Callout type="info">
ภาษาและ filepath

ข้อความหลัง `:` แสดงเป็นป้ายชื่อเท่านั้น ไม่ส่งผลต่อ syntax highlighting ใช้ language identifier ที่ถูกต้องก่อน `:` เพื่อให้ highlighting ทำงานได้อย่างถูกต้อง
</Callout>
